# STT End-of-Turn Detection

A high-performance CPU-only predictive model designed to calculate `p_eot` (the probability that a human conversational turn is over at a given pause). This solution dramatically reduces Mean Response Delay from the naive 1600ms baseline down to **358ms** at a 5.0% false cutoff rate on the full test dataset, achieving a phenomenal **0.979 AUC**.

## Key Highlights
* **Strict Causal Feature Extraction**: Processes only audio from $t = 0$ up to `pause_start`, ensuring zero future-data leakage. 
* **Dual Acoustic Representation**: Combines macro-contextual handcrafted signals (energy drop, voicing density) with a dense **256-dimensional prosody embedding** generated by the `resemblyzer` VoiceEncoder using a causal 1.5s window.
* **L1 Feature Selection Pipeline**: To prevent high-dimensional overfitting on the small dataset, the model utilizes an **L1-regularized Logistic Regression** (Lasso) to surgically distill the 260 raw dimensions down to the ~160 most robust, high-signal features.
* **Robust Decision Making**: The distilled features are piped into a `RandomForestClassifier` to capture complex, non-linear end-of-turn acoustic patterns.

## Evaluation
The model output is strictly calibrated probability arrays (`p_eot`). The evaluation script natively handles the sweeping thresholds, arriving at an optimal operating point of **threshold = 0.50, delay = 250 ms**, producing the final averaged 358ms mean response delay. 

## Project Files
* **`predict.py`**: The inference script that loads the saved pipeline model (`model.pkl`), executes feature extraction on the evaluation dataset, and outputs end-of-turn probabilities to `predictions.csv`.
* **`predictions.csv`**: The generated output file containing the `turn_id`, `pause_index`, and the predicted probability `p_eot`.
* **`train_model.py`**: The training pipeline responsible for executing the 5-Fold GroupKFold Cross Validation, extracting features, optimizing thresholds/delays, fitting the `Pipeline` (StandardScaler -> SelectFromModel -> RandomForestClassifier), and saving the finalized weights.
* **`features_ext.py`**: The core extraction script that respects strict causality boundaries (`[0:pause_start]`). It computes handcrafted acoustic features alongside a 256-d `resemblyzer` prosody embedding.
* **`model.pkl`**: The saved and serialized finalized model weights.
* **`verify_results.py`**: A verification utility used to independently calculate the ROC AUC score directly from `predictions.csv` and the ground truth `labels.csv`.
* **`test_pipeline.py`**: A unit testing script designed to verify the correct model topology (Pipeline), feature dimensions (260), and prediction boundaries.
* **`SUMMARY.html`**: An executive summary detailing final performance metrics and the contents of the submission package.
* **`NOTES.md`**: A technical deep dive outlining CPU constraints, feature engineering specifics, and the distinction between operating point and empirical latency.
* **`RUNLOG.md`**: Historical logs tracking the project's 6 main iterations, from baseline through grid search to final evaluation.
